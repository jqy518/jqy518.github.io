利用hexo搭建github个人博客注意事项

  • blog
  • github
  • hexo
  • 0次

折腾了一天多终于在gitHup上把博客搭建起来了(再展示下成果~~jekion.一路遇到好多坎坷，主要是自己对hexo及yaml格式不熟，现炒现卖，浪费了很多时间；不过最终还是搞定了，现在把这些问题归纳下；希望对想利用hexo在githup上搭建博客的小白能起些参考作用。

部署时遇到的问题，

在_config.yml里添加了githup仓库地址https://github.com/jqy518/jqy518.github.io.git,然后通过hexo d部署，发现没任何输出信息；到仓库去看空空的，显然没有布置成功，百度发现地址的写法有问题改成https://jqy518@github.com/jqy518/jqy518.github.io.git；再部署成功了。

CNAME部署被删除问题

CNAME文件每次重新部署都会被删除，其实只需要把文件拷备到source下面就行了。这样就会一起迁入。

yaml格式问题。

这个是自己不熟习，key与value之间有空格。删除空格会报错。

主题应用问题。

我用的yilia主题感觉很清爽,喜欢。但部署后却报JS错误(可能是自己没有配置对),后查看首页源码发现yiliaConfig对象root属性赋值有问题:root :/报错，/应该用引号包裹（可能是配置有问题）;比较作者的博客，发现他的yiliaConfig对象中无些属性，于是索性暴力一点在ejs模版中删除。再main.js中相关引用直接写死。再测试一切正常。

国际化问题。

yilia 默然是英文版，要在主_config.yml中配置language属性为zh-Hans。
